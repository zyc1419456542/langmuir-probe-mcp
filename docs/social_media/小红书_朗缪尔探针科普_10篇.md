## 第一篇: 一根金属丝如何"看见"电子的能量？

**配图**: `social_01_IV_to_EEPF.png`

**标题**: 一根0.15mm的金属丝，伸进等离子体扫一圈电压，就能看见电子的"能量指纹"

**正文**:

左边是这根钨丝测到的 I-V 曲线——探针偏压从 -40V 扫到 +40V。右边是同一个数据的"魔法变身"——EEPF（电子能量概率分布）。

两个关键电压：
Vf（浮动电位）：电流刚好为零 → 进来的电子和离子恰好平衡
Vp（等离子体电位）：dI/dV 最大 → 电子开始畅通无阻

右边的 EEPF 是怎么来的？对 I-V 曲线求两次导数，套 Druyvesteyn 公式一转——电子按能量分布就出来了。

紫色虚线右边的橙色区域是**高能电子尾巴**。正常等离子体里高能电子很少（~4%）。如果这个比例突然飙到 35%——你的等离子体在"发烧"。

数学上：F(epsilon) ∝ sqrt(epsilon) · d²I/dV²
白话版：**"曲线在某电压处弯得越厉害，该能量的电子就越多"**

## 第二篇: EEPF 的三种"脸"——同一个 Te，三种完全不同的电子结构

**配图**: `eepf_d2v_comparison.png`

**标题**: 你的"电子温度"可能是假的——EEPF 告诉你真相

**正文**:

三个不同状态的等离子体，用同样的探针、同样的方法测出来的 EEPF：

**左列 dI/dV（一阶导）**：
上图 — 平滑单峰，电子乖乖服从麦克斯韦分布
中图 — 出现额外鼓包，有第二群能量不同的电子混进来了
下图 — 末端上翘，高能电子异常增多

**右列 d²I/dV²（二阶导 = EEPF 源函数）**：
Druyvesteyn 公式的原材料。曲线上的每一个鼓包，都对应一群特定能量的电子。

残酷的事实：**这三个状态如果用传统方法测，可能报出"差不多"的电子温度。但它们的电子能量结构完全不同。**

只报一个 Te 数字？你丢失了 90% 的信息。下次看论文看到"电子温度 = X eV"，记得问一句：EEPF 长什么样？

## 第三篇: 探针曲线上的台阶——6 群电子的"身份证"

**配图**: `cath36.png`

**标题**: dI/dV 曲线上有 6 个台阶！仪器坏了？不，这是 6 群不同能量的电子

**正文**:

左图全区（绿虚线 = Vf），右图末端放大。看到了吗？dI/dV 曲线上一级一级的台阶，每个台阶对应导数上的一个峰。

先排查仪器伪影，四项测试全部通过：
1. 台阶只出现在电子排斥区（V > Vf），离子区干干净净
2. 台阶不锁死在任何固定电压——随等离子体状态漂移
3. 不在源表量程切换点
4. 第一个台阶跟着 Vf 走（Vf + 2.21 ± 0.83V）

那这些台阶是什么？
**每越过一个台阶 = 一群特征能量不同的电子开始被探针收集。** 探针偏压逐步升高，不同能量的电子群"逐群解锁"。

这个工况检测到 6 个台阶 → 至少 6 群能量不同的电子。

注意台阶"宽窄不一"（2-8V 不等）——这恰好排除了"均匀间距的数字伪影"。大自然不制造均匀的台阶。

## 第四篇: 台阶对比——为什么低电流时电子群更"分裂"？

**配图**: `cath33.png` `cath01.png`

**标题**: 同样的探针，不同的台阶数——等离子体条件如何改变电子群结构

**正文**:

接上篇，来看两个不同条件下的 dI/dV 台阶形态。

**左图（4台阶）**：数值和视觉计数完全一致，台阶间距不锁固定电压。每个台阶宽度 2-6V——对应着不同电子群的能量展宽。

**右图（数值单峰，视觉判读 3 峰）**：数值和视觉有 ±1-2 个峰的偏差——这就是为什么需要"数值 + 视觉双通道"交叉验证。

统计结果：**低电流组中位台阶数 3，高电流组中位仅 1。** p = 2.5 × 10⁻⁶——这意味着如果两组实际无差异，观察到这个差距的概率小于百万分之二点五。

所以电流越低，电子群越"分裂"；电流越高，电子群越"融合"。

这个现象背后是电子碰撞热化：电流大 → 电子密度高 → 碰撞频繁 → 电子群之间通过碰撞"抹平"差异 → 台阶消失。从一根金属丝的 I-V 曲线上，就能读出等离子体内部的碰撞物理。

## 第五篇: 上翘现象——扫描窗外藏着什么？

**配图**: `cath06.png` `an34.png`

**标题**: 曲线末端突然翘起——扫描窗外有一个未被采集的"第二拐点"

**正文**:

dI/dV 曲线在接近 +40V 扫描上限时末端突然上翘。这是什么？

先排除平滑边界伪影——把末端 15 个点裁掉，上翘还在。不是算法造成的。

**左图（cath06）**：末端清晰回升，数值和视觉一致判定上翘。
**右图（an34）**：平滑上升 + 末端回升，上翘指示扫描窗外可能有第二个拐点。

候选物理机制：
1. 束区高电位第二 knee — 更高能量的电子群在更正的偏压处才被收集
2. 厚鞘膨胀效应 — OML 鞘层膨胀导致收集面积增大
3. 振荡卷积抹开 — 被数据排除（上翘与振荡负相关）

验证方法非常简单：**下一次把扫描上限从 +40V 提高到 +60V**，一个实验就能判定第二拐点是否真的存在。

科学的魅力：数据告诉你"这里有个东西"，但你需要设计新实验去抓住它。

## 第六篇: dI/dV 形态画廊——三种工况，三种"脸"

**配图**: `an30.png` `an60.png`

**标题**: 从平滑鼓包到锐利上翘——dI/dV 形态画廊

**正文**:

三张图，三种完全不同的等离子体状态：

**an30**：平滑单峰鼓包，无分立结构。安静等离子体的典型形态——EEPF 近麦克斯韦。

**an34**：平滑上升 + 末端回升上翘。高能电子开始"冒头"的过渡态。

**an60**：平滑对称鼓包 + 最深的上翘（0.276）。最强磁场下最安静的状态，反而上翘最深。

这是反直觉的发现：我们原以为"越安静越平滑"，但数据告诉我们——**最安静的工况反而有最明显的静态结构**。振荡不是上翘的原因，振荡反而在"抹平"上翘。

如果振荡造成了上翘，那振荡最强的工况上翘应该最深。但实测恰恰相反：上翘深度与振荡强度负相关（Spearman rho = -0.26）。

数据推翻直觉。这就是做实验的意义。

## 第七篇: f_tail——一个数字判断等离子体的"健康状态"

**配图**: `macro_micro_f_tail.png`

**标题**: 你的等离子体"发烧"了吗？——f_tail 参数

**正文**:

定义很简单：
f_tail = 能量高于鞘层势垒的电子数 / 总电子数 × 100%

翻译：**"有资格翻墙的高能电子占比"**。

左边是三组等离子体在参数空间里的分布，右边一画 f_tail 的箱线图：
状态 A：19.5% — 正常，高能电子不多
状态 B：21.2% — 有点不对劲
状态 C：**34.7%** — 明显异常！效应量 d = 2.10（统计学上非常显著）

**f_tail 翻倍了，但鞘层状态比纹丝不动**（4.17 ± 0.23，变异系数仅 5.5%）。

这意味着 f_tail 的变化是真实的电子能量分布重构，不是测量系统偏差。一个数字就能做模式判别的第一道筛子。

重要陷阱——"标量盲区"：
同一个 f_tail ~ 35% 背后可能有两种完全不同的 EEPF 形状——分立多峰束状 vs 连续肥尾。f_tail 适合做筛查，机理分析还是要回到 EEPF 的峰位结构。

-40V 到 +40V，500 个点，27 秒。经过完整的分析管线，提取 **17 个参数**。

**面板解析（6 区全景）**：
A区 dI/dV 曲线族 — 注意两图的差异：左平滑鼓包 vs 右锐利分立台阶

## 第十篇: 从一根金属丝到等离子体全貌——完整分析工具链开源

**配图**: `social_01_IV_to_EEPF.png`

**标题**: 开源！朗缪尔探针全自动分析工具——21 个工具，纯 Python

**正文**:

前面 9 篇讲的这些分析——I-V 曲线处理、EEPF 重建、台阶检测、上翘判别、f_tail 计算、多 AI 审计、视觉验证——全部打包成一个开源 Python 包。

**Langmuir Probe MCP** — GitHub 开源 (MIT 协议)

**21 个工具，覆盖 46 项分析技术**：
- probe_analyze — 单条 IV 全量分析（一键 17 个参数）
- probe_batch — 批量处理 + result.txt
- probe_eepf — Druyvesteyn EEPF 重建
- probe_fit_bimaxwell — Yip 2020 双温迭代拟合
- probe_spectrum — FFT 频谱 + 振荡模式检测
- probe_similarity — 相似参数分析
- probe_predict_mode — 模式概率预测 (88.9%)
- probe_detect_steps + probe_detect_upturn — 台阶/上翘检测
- probe_detect_comb_teeth — 梳齿伪影检测

- probe_gate + probe_gate_multi_ai — 质量闸门 + 多 AI 审计

**零 MATLAB 依赖**，纯 Python (numpy/scipy/matplotlib)。
**支持 Claude Code / Cursor 直接调用**。
